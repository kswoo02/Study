## 📘 객체지향 프로그래밍(OOP) 핵심 정리 – 쉽게 설명한 버전

### 1. 🧊 추상화 (Abstraction)

- **쉽게 말하면?**
    - 복잡한 건 감추고, **필요한 것만 보여주는 것**

- **비유**
    - 자판기 버튼만 누르면 커피가 나와요. 안에 기계가 어떻게 작동하는지는 몰라도 되죠!
        
- **왜 쓰나?**
    - 사용자는 **어떻게 쓰는지만 알면 되니까**
        
- **예시 (Java)**
``` java

`interface Animal {  
	void sound();  // 소리 내는 방식만 약속 
}

``` 

---

### 2. 🧬 상속 (Inheritance)

- **쉽게 말하면?**
    - 부모의 **기능을 자식이 물려받는 것**
        
- **비유**
    - 부모가 운전면허를 갖고 있다면, 자식도 당연히 운전할 수 있다고 생각하는 것
        
- **왜 쓰나?**
    - **같은 기능을 또 안 만들고 재활용**하려고!
        
- **예시 (Java)**
    
``` java    
`class Animal {
	void eat() {   
	    System.out.println("냠냠");
	} 
}

class Dog extends Animal {
	void bark() {
		System.out.println("멍멍");     
	} 
}
```    

---

### 3. 🛡️ 캡슐화 (Encapsulation)

- **쉽게 말하면?**
    
    - **중요한 정보는 숨기고, 필요한 방법만 제공하는 것**
        
- **비유**
    
    - 약병 뚜껑처럼 막아놓고, 정해진 방식(처방전)으로만 먹게 하는 것
        
- **왜 쓰나?**
    
    - 아무나 막 건드리면 **문제가 생기니까 보호**하려고!
        
- **예시 (Java)**
    
    java
    
    복사편집
    
    `class Person {     private String name;  // 직접 접근 X      public String getName() {         return name;     }      public void setName(String name) {         this.name = name;     } }`
    

---

### 4. 🎭 다형성 (Polymorphism)

- **쉽게 말하면?**
    - **같은 명령어**를 줬는데, **다른 방식**으로 동작하는 것
        
- **비유**
    - “연주해!”라고 말했을 때
        - 피아노는 뚱땅
        - 바이올린은 켜는 소리를 낼 수 있어요
            
- **왜 쓰나?**
    
    - **하나의 코드로 여러 상황을 처리**할 수 있어서 유연함
        
- **예시 (Java)**
``` java

Animal a1 = new Dog();
Animal a2 = new Cat();  
a1.sound();  // 멍멍 a2.sound();  // 야옹`

```

## 🧠 핵심 요약 한 줄씩

| 개념  | 한 줄 요약                    |
| --- | ------------------------- |
| 추상화 | 중요한 것만 보여줘서 사용하기 쉽게 만듦    |
| 상속  | 부모의 기능을 자식이 물려받음          |
| 캡슐화 | 중요한 정보는 숨기고, 정해진 방법으로만 접근 |
